## Exercise 1: Review the default IPv6 configuration

### Task 1: Identify the default IPv6 configuration
1. Sign in to LON-DC1 as `Contoso\Administrator` / `Pa55w.rd`.
2. Open Server Manager → Tools → DNS.
3. Expand LON-DC1 → Forward Lookup Zones → Contoso.com.
4. Observe that only LON-DC1 has an IPv6 address and no AAAA records exist for other computers.
5. Close DNS Manager.
6. Open Windows PowerShell ISE.
7. Run: `ipconfig`
8. Record the link-local IPv6 address (starts with FE80::). Do NOT include the % value.
9. Run: `Get-NetIPAddress`
10. Confirm the same link-local IPv6 address appears.
11. Repeat steps 6–10 on LON-SVR1 and TOR-SVR1 (sign in as `Contoso\Administrator` / `Pa55w.rd`).
12. Note: Link-local IPv6 addresses are NOT registered in DNS.

### Task 2: Test link-local address connectivity
1. Switch to LON-DC1.
2. Run: `ping <LON-SVR1-LinkLocal-IPv6>`
3. Confirm four successful replies.
4. Run: `Test-NetConnection <LON-SVR1-LinkLocal-IPv6>`
5. Verify `PingSucceeded : True`.
6. Run: `Test-NetConnection <TOR-SVR1-LinkLocal-IPv6>`
7. Confirm `PingSucceeded : False` and `DestinationHostUnreachable`.
8. Reason: Link-local IPv6 addresses are not routable across subnets.

### Task 3: View the IPv6 routing table
1. On LON-DC1, run: `Start-Process netsh`
2. At the netsh prompt, run:
   - `interface ipv6`
   - `show route`
   - `show interface`
3. Run: `show neighbors`
4. Run: `show destinationcache`
5. Exit netsh.

## Exercise 2: Implement DHCPv6

### Task 1: Create and configure DHCPv6 scopes
1. On LON-DC1, open Server Manager → Tools → DHCP.
2. Expand lon-dc1.contoso.com → IPv6.
3. Right-click IPv6 → New Scope.
4. Name: `Headquarters IPv6`.
5. Prefix: `fd00:0000:0000:0000::`.
6. Exclusion range:
   - Start: `0000:0000:0000:0000`
   - End:   `0000:0000:0000:00ff`
7. Accept default lease duration.
8. Finish the wizard.
9. Close the DHCP console.

### Task 2: Verify IPv6 address assignment
1. Switch to LON-CL1.
2. Sign in as `Contoso\Administrator` / `Pa55w.rd`.
3. Open Windows PowerShell ISE.
4. Run: `ipconfig /renew6`
5. Verify the assigned IPv6 address is within `FD00::/64`.

- Link-local IPv6 connectivity tested and understood.
- IPv6 routing table reviewed.
- DHCPv6 scope created and validated.
