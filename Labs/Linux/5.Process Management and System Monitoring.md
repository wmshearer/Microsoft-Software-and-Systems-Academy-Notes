### Ubuntu Linux System Administration Fundamentals
### Lab: Process Management and System Monitoring

### Overview
This lab installs and runs sample tools that simulate CPU, memory, disk,
and network usage. You will monitor and manage these processes using
standard Linux system-monitoring utilities.

------------------------------------------------------------

### Exercise 1: Simulate and Monitor CPU and Memory Usage

### Objective
Generate CPU and memory load using stress and monitor usage using
top, htop, and free.

- [ ] Update package lists
  Command: `sudo apt update`

- [ ] Install stress and htop
  Command: `sudo apt install stress htop -y`

- [ ] Run stress to simulate CPU and memory load
  Command: `stress --cpu 2 --vm 1 --vm-bytes 512M --timeout 60s`

  --cpu 2        -> 2 CPU stress workers
  --vm 1         -> 1 memory worker
  --vm-bytes 512M-> 512 MB RAM
  --timeout 60s  -> run for 60 seconds

- [ ] Open a second SSH session to the same VM
  Command: `ssh <your-username>@<vm-ip-address>`

- [ ] Monitor processes using top
  Command: `top`

  Notes:
  - Press P to sort by CPU
  - Press M to sort by memory
  - Press q to quit

- [ ] Monitor processes using htop
  Command: `htop`

  Notes:
  - F6 = sort
  - F9 = kill process
  - q  = quit

- [ ] Inspect memory usage
  Command: `free -h`

- [ ] Review manuals
  Commands:
  `man stress`
  `man top`
  `man htop`

------------------------------------------------------------

### Exercise 2: Simulate Disk Activity and Monitor I/O

### Objective
Generate disk activity and analyze disk usage and directory size.

- [ ] Create test directory and enter it
  Command: `mkdir ~/disktest && cd ~/disktest`

- [ ] Generate 50 test files (10MB each)
  Command:
  `for i in {1..50}; do dd if=/dev/zero of=file$i bs=10M count=1; done`

- [ ] Check total disk usage
  Command: `df -h`

- [ ] Check size of test directory
  Command: `sudo du -sh ~/disktest`

- [ ] Delete test directory
  Command: `rm -rf ~/disktest`

- [ ] Return to home directory
  Command: `cd ~`

- [ ] Review manuals
  Commands:
  `man df`
  `man du`

------------------------------------------------------------

### Exercise 3: Generate Network Activity and Monitor

### Objective
Simulate network traffic and inspect connections and bandwidth usage.

- [ ] Install curl and iftop
  Command: `sudo apt install curl iftop -y`

- [ ] Generate network traffic
  Command:
  `for i in {1..20}; do curl https://example.com > /dev/null 2>&1; done`

- [ ] Monitor live bandwidth
  Command: `sudo iftop`

  Notes:
  - Press q to quit

- [ ] View listening ports and sockets
  Command: `ss -tuln`

- [ ] Review manuals
  Commands:
  `man ss`
  `man iftop`

------------------------------------------------------------

### Exercise 4: Analyze Logs for System Events and Stress Tests

### Objective
Inspect system logs for abnormal or resource-heavy behavior.

- [ ] View live system logs
  Command: `sudo journalctl -xe`

  Notes:
  - Look for entries during stress tests
  - Press q to quit

- [ ] View journald service logs
  Command: `sudo journalctl -u systemd-journald`

- [ ] View syslog (if enabled)
  Command: `sudo less /var/log/syslog`

  Controls:
  - Space = scroll
  - /     = search
  - q     = quit

- [ ] Review manuals
  Commands:
  `man journalctl`
  `man syslog`

- [ ] Install Glances
  Command: `sudo apt install glances -y`

- [ ] Run Glances
  Command: `glances`

  Notes:
  - Real-time CPU, memory, disk, network
  - Press q to quit

------------------------------------------------------------

You simulated real CPU, memory, disk, and network workloads,
monitored system performance, inspected logs, and practiced
real-world Linux troubleshooting techniques.
