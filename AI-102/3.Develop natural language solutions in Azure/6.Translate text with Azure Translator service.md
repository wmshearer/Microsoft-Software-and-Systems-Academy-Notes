# Azure Translator

## What it is
Azure Translator is a **text translation API** that supports **90+ languages** and enables:
- Language detection
- One-to-many translation
- Script transliteration (same language, different script)

---

## Azure Translator resource
To use Azure Translator, you must provision a resource in Azure:
- **Standalone Azure Translator resource**, or
- **Text Translation API inside Foundry Tools**

After provisioning, you use:
- **Subscription key**
- **Region**
to authenticate API calls (REST or SDKs).

---

## Core capabilities

### Language detection
- Detects the language of input text
- Returns:
  - Language code (ex: `ja`)
  - Confidence score (0–1)
  - Whether translation/transliteration is supported

Example payload:
`[{ 'Text': 'こんにちは' }]`

---

### Translation
- Converts text **from one language to another**
- Supports:
  - Explicit source language (`from`)
  - One-to-many targets (`to=fr&to=en`)

Example result:
- Japanese → English → French

Key concept:
- **Translate = meaning changes language**

---

### Transliteration
- Converts text **between scripts**, not languages
- Example:
  - Japanese Hiragana → Latin
  - Cyrillic → Latin

Key concept:
- **Transliterate = same language, different script**

---

## Translation options

### Word alignment
- Helps map **source characters to translated characters**
- Useful for languages without spaces (ex: Chinese)
- Enabled with `includeAlignment=true`

---

### Sentence length
- Returns character length of source vs translated text
- Useful for UI layout decisions
- Enabled with `includeSentenceLength=true`

---

### Profanity filtering
Control how profanities are handled:
- `NoAction` → translate normally
- `Deleted` → remove profanities
- `Marked` → obscure profanities
  - Marker options:
    - `Asterisk` (***)
    - `Tag` (XML tags)

---

## Custom Translator

### Why custom translation?
Default models work well generally, but **industry-specific terms** may translate poorly (legal, medical, HR, finance).

### Custom Translator workflow
1. Create workspace in **Custom Translator portal**
2. Link to Azure Translator resource
3. Create project
4. Upload training data
5. Train model
6. Test and publish
7. Use **category ID** in API calls

### Using a custom model
- Pass `category=<category-id>` in translate request
- Overrides default translation model

---

## API essentials

### Translate endpoint
`https://api.cognitive.microsofttranslator.com/translate?api-version=3.0`

Required:
- `to=<language>`
- `Ocp-Apim-Subscription-Key`
- `Content-Type: application/json; charset=UTF-8`

Optional:
- `from=<language>`
- `category=<custom-model-id>`
