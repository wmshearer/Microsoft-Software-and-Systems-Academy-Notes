============================================================
MICROSOFT ENTRA ID — AUTHENTICATION CAPABILITIES (NOTES)
============================================================

OVERVIEW
- Microsoft Entra ID verifies user identities when signing in to devices, apps, or services
- Supports multiple authentication methods to balance security and usability
- Goal: reduce password reliance and move toward passwordless authentication

------------------------------------------------------------
AUTHENTICATION METHODS
------------------------------------------------------------

Passwords
- Most common authentication method
- Weak on their own (single-factor authentication)
- Easy to guess, steal, or reuse
- Best practice: supplement or replace with stronger methods

------------------------------------------------------------

Phone-based authentication
- SMS-based authentication
  - Can be primary authentication (passwordless SMS sign-in)
  - Can be secondary authentication for MFA or SSPR
  - User receives a verification code via text message

- Voice call verification
  - Secondary authentication only
  - Automated call prompts user to press # to verify
  - Not supported as a primary authentication method

------------------------------------------------------------

OATH (Open Authentication)
- Uses time-based one-time passwords (TOTP)
- Two forms:
  - Software OATH tokens (apps)
  - Hardware OATH tokens (key fob–style devices, preview)
- Only supported as secondary authentication
- Used with MFA or Self-Service Password Reset (SSPR)

------------------------------------------------------------
PASSWORDLESS AUTHENTICATION
------------------------------------------------------------

Goal:
- Eliminate passwords from sign-in events
- Reduce phishing and credential theft

Passwordless methods are harder to duplicate or steal and improve security posture

------------------------------------------------------------

Windows Hello for Business
- Passwordless primary authentication
- Uses:
  - Device-bound key or certificate
  - PIN or biometrics (fingerprint / face)
- Two-factor by design:
  - Something you have (device)
  - Something you know (PIN) or are (biometric)
- Protects against credential theft
- Can also be used as secondary MFA method

------------------------------------------------------------

FIDO2 Security Keys
- Open standard for passwordless authentication
- Hardware-based (USB, NFC, Bluetooth)
- Resistant to phishing
- Can be used as:
  - Primary authentication
  - Secondary authentication for MFA
- Supports:
  - Entra ID sign-in
  - Hybrid-joined devices
  - Browser-based authentication

------------------------------------------------------------

Microsoft Authenticator App
- Can be primary or secondary authentication
- Passwordless sign-in flow:
  - User enters username
  - Matches number on screen
  - Confirms with biometric or PIN
- Also supports:
  - Push notifications (Approve/Deny)
  - OATH verification codes
- Available on iOS and Android

------------------------------------------------------------

Certificate-Based Authentication (CBA)
- Uses X.509 certificates
- Primary passwordless authentication
- Based on public key infrastructure (PKI)
- Often used in high-security or regulated environments

------------------------------------------------------------
PRIMARY VS SECONDARY AUTHENTICATION
------------------------------------------------------------

Primary authentication examples:
- Password
- Windows Hello for Business
- FIDO2
- Microsoft Authenticator (passwordless)
- Certificate-based authentication

Secondary authentication examples (MFA / SSPR):
- SMS
- Voice call
- Microsoft Authenticator app
- OATH tokens
- Windows Hello for Business
- FIDO2

------------------------------------------------------------
MULTIFACTOR AUTHENTICATION (MFA)
------------------------------------------------------------

Definition:
- Requires two or more authentication factors

Factors:
- Something you know (password, PIN)
- Something you have (phone, hardware key)
- Something you are (biometrics)

Benefits:
- Dramatically improves security
- Minimal impact on user experience
- Integrated directly into Entra sign-in flow

MFA verification options include:
- Microsoft Authenticator app
- Windows Hello for Business
- FIDO2 security keys
- OATH hardware/software tokens
- SMS
- Voice call

------------------------------------------------------------
SECURITY DEFAULTS
------------------------------------------------------------

Purpose:
- Provide baseline security with no extra cost

Includes:
- Enforced MFA registration for all users
- Mandatory MFA for administrators
- MFA challenges when risk is detected

Best for:
- Small orgs
- Free-tier tenants
- Organizations without P1/P2 licenses

------------------------------------------------------------
SELF-SERVICE PASSWORD RESET (SSPR)
------------------------------------------------------------

Purpose:
- Allows users to reset passwords without IT help
- Reduces help desk calls
- Improves productivity

Requirements:
- Entra ID license
- Enabled by administrator
- User registration with authentication methods

SSPR authentication methods:
- Mobile app notification
- Mobile app code
- Email
- Mobile phone
- Office phone
- Security questions (users only, not admins)

Notes:
- Admin accounts cannot use security questions
- Supports password writeback to on-prem AD
- Email notifications can alert admins of reset events

------------------------------------------------------------
PASSWORD PROTECTION
------------------------------------------------------------

Microsoft Entra Password Protection:
- Blocks weak and commonly used passwords
- Enforced during password change/reset

Global banned password list:
- Maintained by Microsoft
- Updated using real-world attack telemetry
- Automatically applied to all tenants
- Cannot be disabled

Custom banned password list (P1/P2 required):
- Organization-specific terms
- Examples:
  - Company name
  - Product names
  - Locations
  - Internal abbreviations

Protects against:
- Password spray attacks
- Common variations (Passw0rd, P@ssword1, etc.)

------------------------------------------------------------
HYBRID PASSWORD PROTECTION
------------------------------------------------------------

- Extends Entra password protection to on-prem AD
- Requires on-prem components
- Applies same password rules everywhere
- Ensures consistent security across environments

------------------------------------------------------------
MODULE ASSESSMENT — QUICK ANSWERS
------------------------------------------------------------

1) Most secure authentication method:
- Windows Hello

2) MFA-compatible verification methods:
- Microsoft Authenticator app, SMS, Voice call, FIDO2, Windows Hello for Business

3) Feature to reduce IT costs without compromising security:
- Self-service password reset
